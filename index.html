1A. History of the Web

The World Wide Web was invented by Tim Berners-Lee in 1989 while working at CERN.

It was designed to facilitate the sharing of information among researchers.

The first website was launched in 1991, and the web has since evolved into a global platform for communication, commerce, and entertainment.

1B. Client-server architecture

In a client-server architecture, the client (usually a web browser) sends requests to a server, which processes the requests and returns responses.

The server hosts the website's files and handles tasks such as data storage, processing, and security.

This architecture allows for efficient resource management and scalability.

1C. Static vs. dynamic web

Static websites are built with HTML and CSS, and the content is fixed.

Dynamic websites use server-side scripting to generate content on-the-fly. PHP, Python, and Node.js are popular choices for creating dynamic websites.

1D. The difference between the front-end and the back-end

The front-end refers to the part of a website that users interact with directly, including the design, layout, and user interface.

The back-end refers to the server-side of a website, which handles data storage, processing, and business logic.

1E. Website editors, frameworks, and CMS

Website editors are tools that allow users to create and manage website content without needing to write code. Examples : Wix.

Frameworks are pre-built libraries that provide a structure for developing websites. Examples include React, Angular, and Vue.js.

Content Management Systems (CMS) are software applications that allow users to create, manage, and modify website content without needing technical knowledge. Examples include WordPress, Joomla, and Drupal.

2A. HTML: presentation

HTML (HyperText Markup Language) is the standard language for creating web pages. It uses tags to structure content and define elements such as headings, paragraphs, links, and images.

HTML5 is the latest version of HTML, which includes new features such as semantic elements, multimedia support, and improved form controls.

2B. HTML: DOM

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the structure of a document as a tree of objects, allowing developers to manipulate the content and structure of a web page dynamically using JavaScript.

2C. HTML: structuring an HTML page

An HTML page is structured using various elements such as <head>, <body>, <header>, <nav>, <main>, and <footer>. These elements help organize content and improve accessibility.

Example of HTML Structure

<!DOCTYPE html>
<html lang="en">
<!-- the head section is for metadata and links -->
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<!-- the body section is for content that will be displayed on the page -->
<body>
	<!-- the header section typically contains the website's title and navigation -->
    <header>
        <h1>My Website</h1>
    </header>
	<!-- the nav section contains links to different parts of the website -->
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
        </ul>
    </nav>
	<!-- the main section contains the primary content of the page -->
    <main>
        <p>This is the main content of the page.</p>
    </main>
	<!-- the footer section typically contains copyright information and links to legal pages -->
    <footer>
        <p>© 2023 My Website. All rights reserved.</p>
    </footer>
</body>
</html>

2D. HTML: concept of tags

HTML tags are the building blocks of an HTML document. They are used to define the structure and content of a web page.

Tags are typically enclosed in angle brackets, such as <h1> for a heading or <p> for a paragraph. Some tags have attributes that provide additional information about the element, such as <a href="https://example.com"> for a hyperlink.

Example of HTML Tags

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
	<!-- h1 is a heading tag, and p is a paragraph tag -->
    <h1>My Website</h1>
	<!-- p is a paragraph tag that contains text content -->
    <p>This is a paragraph.</p>
</body>
</html>

2E. CSS: presentation

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a web page. It allows developers to control the layout, colors, fonts, and overall appearance of a website.

CSS3 is the latest version of CSS, which includes new features such as animations, transitions, and improved layout capabilities.

2F. CSS: selector types

CSS selectors are used to target HTML elements and apply styles to them. Common selector types include element selectors (e.g., p), class selectors (e.g., .class-name), and ID selectors (e.g., #id-name).

Example of CSS Selectors

/* element selector */
p {
    color: blue;
}

/* class selector */
.class-name {
    font-weight: bold;
}

/* ID selector */
#id-name {
    background-color: yellow;
}

2G. CSS: properties

CSS properties define the specific styles to be applied to elements. Examples include color, font-size, margin, and padding.

Example of CSS Properties

p {
	color: blue; /* sets the text color to blue */
	font-size: 16px; /* sets the font size to 16 pixels */
	margin: 10px; /* sets the margin around the element to 10 pixels */
	padding: 5px; /* sets the padding inside the element to 5 pixels */
}

2H. CSS: responsive design

Responsive design is an approach to web design that ensures a website looks and functions well on various devices and screen sizes. It typically involves using flexible layouts, media queries, and scalable images to create a seamless user experience across different platforms.

Example of Responsive Design with CSS

/* Base styles for all devices */
body {
	font-family: Arial, sans-serif;
}

.container {
	width: 100%;
	padding: 20px;
}

/* Styles for screens larger than 600px */
@media (min-width: 600px) {
	.container {
		width: 80%;
		margin: 0 auto;
	}
}

3. The basics of JavaScript development

  • A. JavaScript: basic concepts (variables, data types, control structures)
  • B. JavaScript: functions (presentation, variable scope, anonymous functions)
  • C. JavaScript: events and DOM manipulation

3A. JavaScript: basic concepts

JavaScript is a programming language used to create interactive and dynamic web content. It allows developers to manipulate the DOM, handle events, and perform various tasks on the client-side.

Basic concepts in JavaScript include variables (used to store data), data types (such as strings, numbers, and booleans), and control structures (like if statements and loops).

3B. JavaScript: functions

Functions in JavaScript are reusable blocks of code that perform a specific task. They can take parameters and return values. Variable scope refers to the accessibility of variables within different parts of the code, while anonymous functions are functions without a name, often used as callbacks.

3C. JavaScript: events and DOM manipulation

JavaScript allows developers to handle events (such as clicks, mouse movements, and keyboard input) and manipulate the DOM to create dynamic web pages. This can be done using methods like addEventListener and properties like innerHTML.

4. Advanced JavaScript

  • A. JavaScript objects
  • B. JSON: Objects and files
  • C. Arrays of objects
  • D. Storage: localStorage and SessionStorage

4A. JavaScript objects

JavaScript objects are collections of key-value pairs that can represent complex data structures. They can contain properties and methods, allowing developers to model real-world entities and behaviors.

4B. JSON: Objects and files

JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy for humans to read and write, and easy for machines to parse and generate. It is commonly used for transmitting data between a server and a web application.

4C. Arrays of objects

Arrays of objects are a common data structure in JavaScript, where an array contains multiple objects. This allows developers to store and manipulate collections of related data efficiently.

4D. Storage: localStorage and SessionStorage

localStorage and SessionStorage are web storage APIs that allow developers to store data in the browser. localStorage provides persistent storage that remains even after the browser is closed, while SessionStorage provides temporary storage that is cleared when the browser session ends.
